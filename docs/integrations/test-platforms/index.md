# Infrastructure providers

Test platforms integrations allow you to connect ReportPortal with external test management and test automation tools where your test cases are created and executed. This integration ensures that test execution results are automatically sent to ReportPortal, providing a centralized view of testing activity across different platforms.

By connecting a test platform, teams can synchronize test cases, execution runs, and results without manual export or import. Each test run is automatically reflected as a launch in ReportPortal, making it easier to track execution history, analyze failures, and monitor overall test progress.

These integrations typically support configuration of key parameters such as the target ReportPortal instance, project name, authentication credentials, and launch naming strategy. This allows teams to ensure that test data is correctly routed and consistently organized across projects.

In addition, test platforms can pass execution metadata and hierarchical test structure, enabling more detailed reporting inside ReportPortal. Depending on the integration settings, results can include step-level or nested step-level information, giving teams flexibility in how deeply they want to analyze test execution flows.

Once configured, every new test execution in the external platform is automatically reflected in ReportPortal. This eliminates manual reporting efforts and ensures that QA teams always have up-to-date visibility into test results.

Overall, test platform integrations help unify test creation, execution, and reporting into a single workflow, improving traceability, reducing overhead, and enabling faster feedback cycles across the QA process.
