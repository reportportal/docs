# Test executions

On the [Test Executions](/features/TestExecutions) page, you can conveniently view and manage individual test cases across multiple launches without navigating through hierarchical launch structures.

## Page layout

At the top of the page, the following launch filtering options are available:

* **Launch Start Time:** Filter by launch start date within predefined ranges (e.g., last 7 days).
* **Launch Names:** Refine results based on specific launch names.
* **Launch Attributes:** Add applicable attributes to customize the filter criteria.

### Header Elements

The following elements are displayed in the header of the page:

* **Export:** Export the filtered table data as a downloadable file.
* **Refresh:** Refresh the table to include the latest updates.
* **All Filters** (icon): Expand to view additional filter options.

### No launches

For a new project, or if no launches have been reported yet, "No results" is displayed.

<MediaViewer src={require('./img/NoResults.png')} alt="Test Executions page when no launches" />

### Launches reported

In case there are launches which correspond default filter criterion (start time = Last 7 days), then the Search Results table displays detailed information about test executions based on applied filters.

Statistics section **in the table header** includes the following counters:

* **Total:** Total number of test executions.
* **Passed:** Count of successful tests.
* **Failed:** Count of failed tests.
* **Skipped:** Count of skipped tests.
* **In Progress:** Count of tests that are currently running.
* **Interrupted:** Count of tests that were prematurely stopped.

The table includes the following **default columns** for each test execution:

* "Name" column
* "Retries" column
* "Start Time" column
* "Status" column
* "Defect Type" column
* "Defect Comment" column

<MediaViewer src={require('./img/LaunchesReported.png')} alt="Launches Reported on the Test Executions page" />

You can expand or collapse all rows in the Search Results table by clicking on the corresponding icon.

In addition, you can expand or collapse an individual row as needed.

<MediaViewer src={require('./img/Expand.png')} alt="Expand or collapse rows on our test reporting tool" />

:::note
The limit for the search results is 10.000 latest test cases.
:::

:::important
Total statistics represent all test results that match the applied filter criteria, irrespective of the 10k item limit on the Test Executions page.
:::

## Launch level filtering

You can filter test executions by the characteristics of the launch to which the tests belong. You can use one or more criteria.

### Launch start time

You can filter test executions by Launch start time using dropdown.

In this case only tests which belong to the launches started in the specified range will be displayed regardless the launch status.

:::note
Test of the launches in "In Progress" status will also be displayed.
:::

Available options:

* Today
* Last 2 Days
* Last 3 Days
* Last 7 Days
* Last 30 Days
* Last 60 Days
* Last 90 Days
* Custom range (date picker)

<MediaViewer src={require('./img/LaunchStartTime.png')} alt="Launch start time filter" />

### Launch names

You can filter test executions by Launch names. In this case, only tests which belong to specified launches will be displayed.

<MediaViewer src={require('./img/LaunchNames.png')} alt="Launch names filter" />

### Launch attributes

You can filter test executions by one or more launch attributes. Only tests which belong to launches with all the specified attributes will be displayed.

:::note
You can filter launch attributes in three ways: by **key**, by **value**, or by a **key:value** pair.
:::

<MediaViewer src={require('./img/LaunchAttributes1.png')} alt="Filtering launches by value" />

## Saved filters

The Saved Filters feature allows users to store filter configurations on the Test Executions page so they can quickly switch between frequently used criteria without reconfiguring filters each time.

This filters are saved per user per project. Other users won't see the filters created by other users.

Users can:

* Create new filters
* Apply saved filters
* Rename saved filters
* Edit saved filters
* Duplicate saved filters
* Delete saved filters
* Reorder saved filters
* Clear applied filters

### Create filter

1. Open the Test Executions page.
2. Click the "All Filters" icon in the header.
3. Configure filter criteria in the "Filters" side panel.
4. Click the "Save Filter" button.

<MediaViewer src={require('./img/saved-filters/CreateFilter1-1.png')} alt="Save Filter button" />

5. If no filters exist, enter filter name (should be unique per user per project).

<MediaViewer src={require('./img/saved-filters/CreateFilter1-2.png')} alt="Save Filter if no filter exist" />

6. If filters exist, choose "Save as new" and provide filter name.

<MediaViewer src={require('./img/saved-filters/CreateFilter2.png')} alt="Save Filter if filters exist" />

7. Click "Save & Apply" button.

### Apply filter

1. Open the Test Executions page.
2. Click the "All Filters" icon in the header.
3. Click the three-dots menu on a saved filter.
4. Select "Apply" from the dropdown.

<MediaViewer src={require('./img/saved-filters/ApplyFilter1.png')} alt="Apply option in the dropdown" />

or

1. Open the Test Executions page.
2. Click the "All Filters" icon in the header.
3. Click on saved filter.
4. Click "Apply" button.

<MediaViewer src={require('./img/saved-filters/ApplyFilter2.png')} alt="Apply button" />

On Test Executions page you will see the name of the filter which is currently applied to the search results.

<MediaViewer src={require('./img/saved-filters/ApplyFilter3.png')} alt="Saved filter on Test Executions page on our qa automation dashboard" />

### Rename filter

1. Open the Test Executions page.
2. Click the "Filters" icon in the header.

## Customize columns

You can customize the columns displayed on the Test Execution page by adding new columns from a predefined list, removing existing columns, and reordering them.

### Add New Column

1. Open Test Executions page.
2. Click the "Customize Columns" button.
3. Click "Add Column" button.
4. Select a column from the dropdown and confirm.
5. Click the "Save" button.

<MediaViewer src={require('./img/AddColumn1.png')} alt="Add column on the Customize Columns filter" />

<MediaViewer src={require('./img/AddColumn2.png')} alt="Options on the Customize Columns filter" />

### Remove Column

1. Open Test Executions page.
2. Click the "Customize Columns" button.
3. Click the "X" button next to a column name.
4. Click the "Save" button.

<MediaViewer src={require('./img/RemoveColumn.png')} alt="Remove column on the Customize Columns filter" />

### Reorder Columns

1. Open Test Executions page.
2. Click the "Customize Columns" button.
3. Drag and drop a column name to reorder the list.
4. Click the "Save" button.

<MediaViewer src={require('./img/ReorderColumns.png')} alt="Reordering columns on the Customize Columns filter" />

### Cancel Changes

1. Open Test Executions page.
2. Click the "Customize Columns" button.
3. Make some changes (for example, add new column).
4. Click the "Cancel" button.

<MediaViewer src={require('./img/CancelChanges.png')} alt="Cancel changes on the Customize Columns filter" />

### Reset Columns

1. Open Test Executions page.
2. Click the "Customize Columns" button.
3. Make some changes (for example, add new column).
4. Click the "Reset to Default" button.
5. Click the "Save" button.

<MediaViewer src={require('./img/ResetToDefault.png')} alt="Reset to default on the Customize Columns filter" />

### Add custom column

1. Open Test Executions page.
2. Click the "Customize Columns" button.
3. Click the "Add Column" button.
4. Select "Custom attribute" option from the dropdown.
5. Specify the key.
6. Click the "Tick" button.

<MediaViewer src={require('./img/CustomColumn1.png')} alt="Selecting Custom attribute" />

7. Click the "Save" button.

<MediaViewer src={require('./img/CustomColumn2.png')} alt="Displaying of custom column on Customize Columns filter" />

The attribute key becomes the column header, and the attribute values are populated for each item.

:::note
You can create multiple "Custom attribute" columns.
:::

## Refresh

You can update the list of test executions without reconfiguring filters.

When new launches or test items that meet the active filter conditions (Launch level and Test level) are reported, using the "Refresh" button updates the Search results table to reflect the most recent data.

To reload data on the Test Executions page:

1. Open Test Executions page.
2. Click the "Refresh" button.

<MediaViewer src={require('./img/Refresh.png')} alt="Refresh option on Test Execution page" />
