# Getting Started

This section helps you quickly get started with ReportPortal and understand how to use it in real testing workflows. It brings together practical tutorial and feature explanations so you can move from initial setup to active usage step by step.
Here you will find clear guides that explain how to connect your test runs, upload results, and begin working with them inside ReportPortal. The focus is on real first actions, helping you understand the basic flow of working with the platform and avoid common setup and usage issues.

A major part of this section is dedicated to how ReportPortal works and what you can do with it in everyday testing. The platform is designed to make test result management more structured and easier to interpret. It collects all test executions in one place, so you can clearly see what is happening across different runs and keep full visibility over your testing activity.

Instead of manually digging through logs, ReportPortal helps you organize results in a more efficient way. It automatically groups similar failures and helps you spot repeating issues, which makes it easier to understand what is actually breaking and where to focus your attention. This reduces the time spent on analysis and speeds up the debugging process.

You will also learn how reporting works in real time. Results are displayed as tests are executed, so you can immediately see what is passing or failing. Visuals like dashboards help you monitor progress, review trends, and quickly understand the current state of your test runs without extra effort.

To see ReportPortal’s features in practice, check out our [demo instance](https://demo.reportportal.io/ui/#login).

Overall, the Getting Started section gives you a simple and practical starting point for working with ReportPortal and helps you understand how to apply it in real testing scenarios.
