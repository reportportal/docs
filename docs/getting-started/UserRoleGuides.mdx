# User role guides

In ReportPortal, permissions and capabilities are defined by a combination of account type and user role. These are organized into two main levels: organization level and project level.

All accounts in ReportPortal are divided into two types according to **instance-level classification**:

* Administrator
* Regular user

An account with the "Administrator" type has permission to configure the entire instance and access all projects.

**Organization-Level Roles**

These roles define permissions across the entire organization:

* Manager
* Member

**Project-Level Roles**

These roles apply within the scope of a specific project:

* Viewer
* Editor

ReportPortal supports different roles across QA, DevOps, and management workflows. Choose your role to get started quickly.
